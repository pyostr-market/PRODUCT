# Спецификация API: Система отзывов товаров

**Версия:** 1.0
**Дата:** 7 апреля 2026 г.
**Статус:** Реализовано

---

## 1. Общие положения

Система отзывов позволяет авторизованным пользователям оставлять отзывы о товарах, содержащие оценку (рейтинг от 1 до 5), текст отзыва и опциональные изображения. Отзывы влияют на рейтинг товара, который отображается в каталоге.

### 1.1 Модель отзыва

Отзыв содержит:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | integer | Уникальный идентификатор отзыва |
| `product_id` | integer | ID товара, для которого оставлен отзыв |
| `user_id` | integer | ID пользователя, оставившего отзыв |
| `username` | string | Имя пользователя (фиксируется на момент создания из JWT) |
| `rating` | decimal | Оценка от 1.0 до 5.0 с шагом 0.1 |
| `text` | string | Текст отзыва (опционально, макс. 5000 символов) |
| `images` | array | Список изображений (опционально) |

### 1.2 Ограничения

- Только **авторизованные** пользователи могут создавать отзывы
- Один пользователь может оставить **только один отзыв** на один товар
- Отзыв может содержать только рейтинг (без текста и изображений)
- Редактировать и удалять отзыв может **только его владелец**
- Рейтинг товара пересчитывается автоматически при каждом операции с отзывами

### 1.3 Рейтинг товара

Рейтинг товара представляет собой объект:

| Поле | Тип | Описание |
|------|-----|----------|
| `value` | float | Среднее значение всех отзывов (0.0 – 5.0), `null` если отзывов нет |
| `count` | integer | Количество отзывов, на основе которых рассчитан рейтинг |

**Формула расчёта:** `value = SUM(rating) / COUNT(reviews)`

Рейтинг товара возвращается в API товаров в поле `rating`.

### 1.4 Изображения в отзывах

Изображения загружаются через Upload API (`POST /uploads/`), после чего `upload_id` передаётся при создании отзыва.

| Поле | Тип | Описание |
|------|-----|----------|
| `upload_id` | integer | ID загруженного изображения из UploadHistory |
| `image_url` | string | Публичный URL изображения |
| `ordering` | integer | Порядок сортировки изображений |

## 2. API Методы

### 2.1 Создать отзыв

**Метод:** `POST /reviews`

**Права доступа:** Требуется авторизация (любой авторизованный пользователь)

#### Запрос

**Content-Type:** `multipart/form-data`

**Параметры:**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `product_id` | integer | Да | ID товара |
| `rating` | decimal | Да | Оценка (1.0 – 5.0) |
| `text` | string | Нет | Текст отзыва (макс. 5000 символов) |
| `images_json` | string | Нет | JSON-массив объектов `{upload_id, ordering}` |

**Формат `images_json`:**

```json
[
  {"upload_id": 10, "ordering": 0},
  {"upload_id": 11, "ordering": 1}
]
```

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "product_id": 100,
    "user_id": 42,
    "username": "Иван Петров",
    "rating": "4.5",
    "text": "Отличный товар, рекомендую!",
    "images": [
      {
        "upload_id": 10,
        "image_url": "https://cdn.example.com/reviews/img-1.jpg",
        "ordering": 0
      },
      {
        "upload_id": 11,
        "image_url": "https://cdn.example.com/reviews/img-2.jpg",
        "ordering": 1
      }
    ]
  }
}
```

**Коды ошибок:**

| Код | Описание |
|-----|----------|
| 400 | Пользователь уже оставлял отзыв на этот товар |
| 401 | Неавторизованный запрос |
| 422 | Некорректные данные (рейтинг вне диапазона, текст слишком длинный) |

    "user_id": 42,
    "username": "Иван Петров",
    "rating": "4.5",

### 2.4 Обновить отзыв

**Метод:** `PUT /reviews/{review_id}`

**Права доступа:** Только владелец отзыва

#### Запрос

**Параметры пути:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `review_id` | integer | ID отзыва для обновления |

**Content-Type:** `multipart/form-data`

**Параметры:**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `rating` | decimal | Нет | Новая оценка (1.0 – 5.0) |
| `text` | string | Нет | Новый текст отзыва |
| `images_json` | string | Нет | JSON-массив операций с изображениями |

**Формат `images_json` (операции):**

```json
[
  {"action": "pass", "upload_id": 10, "ordering": 0},
  {"action": "delete", "upload_id": 11},
  {"action": "create", "upload_id": 12, "ordering": 1}
]
```

| `action` | Описание |
|----------|----------|
| `pass` | Оставить изображение без изменений (можно обновить `ordering`) |
| `delete` | Удалить изображение |
| `create` | Добавить новое изображение (требуется `upload_id`) |

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "product_id": 100,
    "user_id": 42,
    "username": "Иван Петров",
    "rating": "5.0",
    "text": "Обновлённый отзыв — товар действительно отличный!",
    "images": [
      {
        "upload_id": 10,
        "image_url": "https://cdn.example.com/reviews/img-1.jpg",
        "ordering": 0
      },
      {
        "upload_id": 12,
        "image_url": "https://cdn.example.com/reviews/img-3.jpg",
        "ordering": 1
      }
    ]
  }
}
```

**Коды ошибок:**

| Код | Описание |
|-----|----------|
| 401 | Неавторизованный запрос |
| 403 | Недостаточно прав (не владелец отзыва) |
| 404 | Отзыв не найден |
| 422 | Некорректные данные |

## 3. Примеры использования

### 3.1 Создание отзыва с рейтингом и текстом

```bash
curl -X POST http://api.example.com/reviews \
  -H "Authorization: Bearer <token>" \
  -F "product_id=100" \
  -F "rating=4.5" \
  -F "text=Отличный товар, рекомендую!"
```

### 3.2 Создание отзыва с изображениями

```bash
# Сначала загружаем изображения
UPLOAD_1=$(curl -X POST http://api.example.com/uploads \
  -F "file=@photo1.jpg" \
  -F "folder=reviews" \
  -F "original_filename=photo1.jpg")
# {"upload_id": 10, ...}

# Создаём отзыв с изображениями
curl -X POST http://api.example.com/reviews \
  -H "Authorization: Bearer <token>" \
  -F "product_id=100" \
  -F "rating=5" \
  -F "text=Супер!" \
  -F 'images_json=[{"upload_id": 10, "ordering": 0}]'
```

### 3.3 Получение отзывов товара

```bash
curl -X GET "http://api.example.com/reviews/product/100?limit=10&offset=0"
```

**Ответ:**
```json
{
  "success": true,
  "data": {
    "total": 115,
    "average_rating": 4.7,
    "items": [ ... ]
  }
}
```

### 3.4 Обновление отзыва

```bash
curl -X PUT http://api.example.com/reviews/1 \
  -H "Authorization: Bearer <token>" \
  -F "rating=5" \
  -F "text=Обновлённый отзыв — товар действительно отличный!"
```

### 3.5 Удаление отзыва

```bash
curl -X DELETE http://api.example.com/reviews/1 \
  -H "Authorization: Bearer <token>"
```

## 4. Структура базы данных

### Таблица: `reviews`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | BIGINT | Первичный ключ |
| `product_id` | BIGINT | FK на products.id (ON DELETE CASCADE) |
| `user_id` | BIGINT | ID пользователя (без FK) |
| `username` | VARCHAR(255) | Имя пользователя (фиксируется при создании) |
| `rating` | NUMERIC(2,1) | Оценка (1.0 – 5.0) |
| `text` | TEXT | Текст отзыва (nullable, макс. 5000 символов) |
| `created_at` | TIMESTAMP | Дата создания |
| `updated_at` | TIMESTAMP | Дата обновления |

**Уникальные ограничения:**
- `UNIQUE (product_id, user_id)` — один отзыв на товар от пользователя

**Индексы:**
- `IX reviews_product_id`
- `IX reviews_user_id`
- `IX reviews_product_user` (unique composite)

### Таблица: `review_images`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | BIGINT | Первичный ключ |
| `review_id` | BIGINT | FK на reviews.id (ON DELETE CASCADE) |
| `upload_id` | BIGINT | FK на upload_history.id (ON DELETE RESTRICT) |
| `ordering` | INTEGER | Порядок сортировки (по умолчанию: 0) |
| `created_at` | TIMESTAMP | Дата создания |
| `updated_at` | TIMESTAMP | Дата обновления |

### Таблица: `review_audit_logs`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | BIGINT | Первичный ключ |
| `review_id` | BIGINT | FK на reviews.id (ON DELETE CASCADE) |
| `action` | VARCHAR(50) | Действие (create, update, delete) |
| `old_data` | JSON | Старые данные (nullable для create) |
| `new_data` | JSON | Новые данные (nullable для delete) |
| `user_id` | BIGINT | ID пользователя, выполнившего действие |
| `fio` | VARCHAR(255) | ФИО пользователя |
| `created_at` | TIMESTAMP | Дата записи |
| `updated_at` | TIMESTAMP | Дата обновления |

**Индексы:**
- `IX review_audit_logs_review_id`
- `IX review_audit_logs_user_id`
- `IX review_audit_logs_action`

### Таблица: `products` (изменения)

| Поле | Тип | Описание |
|------|-----|----------|
| `rating` | NUMERIC(2,1) | Средний рейтинг товара (nullable) |

## 5. Сценарии использования

### 5.1 Отображение рейтинга в карточке товара

```
GET /product/100
```

В ответе поле `rating` содержит `{value: 4.7, count: 115}`.

### 5.2 Блок отзывов в карточке товара

```
GET /reviews/product/100?limit=10&offset=0
```

Отображаются 10 последних отзывов со средним рейтингом.

### 5.3 Фильтрация каталога по рейтингу

Клиент может использовать поле `rating.value` для фильтрации товаров:

```
GET /product?min_rating=4.0
```

(Фильтрация на стороне клиента по полю `rating.value`)

### 5.4 Личный кабинет — мои отзывы

Пользователь может получить свои отзывы через audit логи:

```
GET /reviews/admin/audit?user_id=42
```

## 6. Примечания

1. **Сортировка отзывов:** Отзывы возвращаются в порядке убывания даты создания (`created_at DESC`) — новые первыми.
2. **Пагинация:** Используется offset-пагинация с параметрами `limit` и `offset`.
3. **Каскадное удаление:** При удалении товара все его отзывы удаляются автоматически (CASCADE).
4. **Каскадное удаление изображений:** При удалении отзыва все его изображения удаляются (CASCADE). При удалении загрузки изображения отзыв не удаляется (RESTRICT на upload_history).
5. **Пересчёт рейтинга:** Рейтинг товара пересчитывается автоматически при создании, обновлении или удалении отзыва. Формула: `AVG(review.rating)` по всем отзывам товара.
6. **Audit-логирование:** Все операции создания, обновления и удаления записываются в `review_audit_logs` с указанием old_data и new_data.
7. **Username фиксируется:** Имя пользователя сохраняется из JWT на момент создания отзыва и не изменяется при последующих обновлениях имени пользователя.
