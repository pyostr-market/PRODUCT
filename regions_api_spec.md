# Спецификация API: Регионы (Regions)

**Версия:** 1.0
**Дата:** 8 апреля 2026 г.
**Статус:** Реализовано

---

## 1. Общие положения

Модуль «Регионы» предоставляет CRUD-операции для управления географическими регионами с поддержкой иерархической структуры (родительский → дочерний регион). Регионы могут быть привязаны к товарам для фильтрации каталога по географическому признаку.

### 1.1 Модель региона

Регион содержит:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | integer | Уникальный идентификатор региона |
| `name` | string | Название региона (уникальное, мин. 2 символа) |
| `parent_id` | integer \| null | ID родительского региона (для иерархии) |
| `created_at` | datetime | Дата создания |
| `updated_at` | datetime | Дата последнего обновления |

### 1.2 Иерархия регионов

Регионы поддерживают неограниченную вложенность через поле `parent_id`:

| Уровень | Пример | `parent_id` |
|---------|--------|-------------|
| 1 | «Россия» | `null` |
| 2 | «Северо-Западный» | ID «Россия» |
| 3 | «Санкт-Петербург» | ID «Северо-Западный» |

При удалении родительского региона дочерние регионы удаляются каскадно (CASCADE).

### 1.3 Связь с товарами

Товар может быть привязан к одному региону через поле `region_id`. Это позволяет:
- Фильтровать товары по региону в каталоге
- Отображать регион в карточке товара
- Выгружать каталог с указанием региона

## 2. API Методы

### 2.1 Создать регион

**Метод:** `POST /region`

**Права доступа:** Требуется permission `region:create`

#### Запрос

**Content-Type:** `application/json`

**Тело запроса:**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `name` | string | Да | Название региона (мин. 2 символа) |
| `parent_id` | integer | Нет | ID родительского региона |

**Пример:**

```json
{
  "name": "Северо-Западный",
  "parent_id": null
}
```

```json
{
  "name": "Санкт-Петербург",
  "parent_id": 1
}
```

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Северо-Западный",
    "parent_id": null
  }
}
```

**Коды ошибок:**

| Код | Описание |
|-----|----------|
| 400 | Некорректные данные (имя слишком короткое) |
| 403 | Недостаточно прав |
| 409 | Регион с таким именем уже существует |

---

### 2.2 Получить регион по ID

**Метод:** `GET /region/{region_id}`

**Права доступа:** Требуется permission `region:view`

#### Запрос

**Параметры пути:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `region_id` | integer | ID региона |

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Северо-Западный",
    "parent_id": null
  }
}
```

**Коды ошибок:**

| Код | Описание |
|-----|----------|
| 404 | Регион не найден |
| 403 | Недостаточно прав |

---

### 2.3 Получить список регионов

**Метод:** `GET /region`

**Права доступа:** Требуется permission `region:view`

#### Запрос

**Query-параметры:**

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| `name` | string | Нет | Фильтр по имени (частичное совпадение, ILIKE) |
| `parent_id` | integer | Нет | Фильтр по родительскому региону |
| `limit` | integer | Нет | Количество элементов (по умолчанию: 10, макс: 100) |
| `offset` | integer | Нет | Смещение (по умолчанию: 0) |

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "total": 2,
    "items": [
      {
        "id": 1,
        "name": "Северо-Западный",
        "parent_id": null
      },
      {
        "id": 2,
        "name": "Санкт-Петербург",
        "parent_id": 1
      }
    ]
  }
}
```

### 2.5 Обновить регион

**Метод:** `PUT /region/{region_id}`

**Права доступа:** Требуется permission `region:update`

#### Запрос

**Параметры пути:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `region_id` | integer | ID региона для обновления |

**Content-Type:** `application/json`

**Тело запроса (частичное обновление):**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `name` | string | Нет | Новое название региона |
| `parent_id` | integer | Нет | Новый родительский регион (или `null`) |

**Пример:**

```json
{
  "name": "Северо-Западный федеральный округ"
}
```

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Северо-Западный федеральный округ",
    "parent_id": null
  }
}
```

**Коды ошибок:**

| Код | Описание |
|-----|----------|
| 400 | Некорректные данные |
| 403 | Недостаточно прав |
| 404 | Регион не найден |

---

### 2.7 Аудит регионов (Admin)

**Метод:** `GET /region/admin/audit`

**Права доступа:** Требуется permission `region:audit`

#### Запрос

**Query-параметры:**

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| `region_id` | integer | Нет | Фильтр по ID региона |
| `user_id` | integer | Нет | Фильтр по ID пользователя |
| `action` | string | Нет | Фильтр по действию (create, update, delete) |
| `limit` | integer | Нет | Количество элементов (по умолчанию: 20, макс: 100) |
| `offset` | integer | Нет | Смещение (по умолчанию: 0) |

#### Ответ

**Успешный ответ (200 OK):**

```json
{
  "success": true,
  "data": {
    "total": 1,
    "items": [
      {
        "id": 1,
        "region_id": 1,
        "action": "update",
        "old_data": {"name": "Северо-Западный", "parent_id": null},
        "new_data": {"name": "Северо-Западный федеральный округ", "parent_id": null},
        "user_id": 42,
        "fio": "Иван Петров",
        "created_at": "2026-04-08T10:20:00"
      }
    ]
  }
}
```

## 3. Регион в API товаров

### 3.1 Создание товара с регионом

**Метод:** `POST /product`

**Content-Type:** `multipart/form-data`

Добавлен параметр:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `region_id` | integer | Нет | ID региона для привязки товара |

### 3.2 Обновление товара с регионом

**Метод:** `PUT /product/{product_id}`

**Content-Type:** `multipart/form-data`

Добавлен параметр:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `region_id` | integer | Нет | Новый ID региона (или `null` для отвязки) |

### 3.3 Ответы API товаров содержат регион

Во всех методах GET для товаров (`GET /product/{id}`, `GET /product`, `POST /product`, `PUT /product/{id}`, поиск) добавлено поле:

| Поле | Тип | Описание |
|------|-----|----------|
| `region` | object \| null | Объект региона с полями `id`, `name`, `parent_id` |

**Пример:**

```json
{
  "success": true,
  "data": {
    "id": 100,
    "name": "iPhone 17 Pro",
    "price": "99990.00",
    "region": {
      "id": 2,
      "name": "Санкт-Петербург",
      "parent_id": 1
    },
    "category": {...},
    "supplier": {...}
  }
}
```

### 3.4 Фильтрация товаров по региону

**Метод:** `GET /product`

Добавлен query-параметр:

| Параметр | Тип | Обязательное | Описание |
|----------|-----|--------------|----------|
| `region_id` | integer | Нет | Фильтр товаров по региону |

**Пример:**

```
GET /product?region_id=2
```

### 3.5 Экспорт каталога с регионом

**Метод:** `GET /product/admin/export`

В ответе экспорта добавлено поле:

| Поле | Тип | Описание |
|------|-----|----------|
| `region` | object \| null | Объект региона с полями `id`, `name` |

**Пример:**

```json
{
  "success": true,
  "data": {
    "total": 1,
    "items": [
      {
        "id": 100,
        "name": "iPhone 17 Pro",
        "price": "99990.00",
        "region": {
          "id": 2,
          "name": "Санкт-Петербург"
        },
        "category": {...},
        "supplier": {...},
        "attributes": [...]
      }
    ]
  }
}
```

## 4. Примеры использования

### 4.1 Создание иерархии регионов

```bash
# Создаём корневой регион
curl -X POST http://api.example.com/region \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "Россия"}'

# Создаём дочерний регион
curl -X POST http://api.example.com/region \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "Северо-Западный", "parent_id": 1}'

# Создаём город
curl -X POST http://api.example.com/region \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "Санкт-Петербург", "parent_id": 2}'
```

### 4.2 Получение дочерних регионов

```bash
curl -X GET http://api.example.com/region/2/children \
  -H "Authorization: Bearer <token>"
```

**Ответ:**
```json
{
  "success": true,
  "data": [
    {"id": 3, "name": "Санкт-Петербург", "parent_id": 2},
    {"id": 4, "name": "Ленинградская область", "parent_id": 2}
  ]
}
```

### 4.3 Создание товара с привязкой к региону

```bash
curl -X POST http://api.example.com/product \
  -H "Authorization: Bearer <token>" \
  -F "name=iPhone 17 Pro" \
  -F "price=99990" \
  -F "region_id=3"
```

### 4.4 Фильтрация товаров по региону

```bash
curl -X GET "http://api.example.com/product?region_id=3" \
  -H "Authorization: Bearer <token>"
```

## 5. Структура базы данных

### Таблица: `regions`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | BIGINT | Первичный ключ, autoincrement |
| `name` | VARCHAR(150) | Название региона (уникальное, NOT NULL) |
| `parent_id` | BIGINT | FK на regions.id (ON DELETE CASCADE, nullable) |
| `created_at` | TIMESTAMP WITH TIME ZONE | Дата создания |
| `updated_at` | TIMESTAMP WITH TIME ZONE | Дата обновления |

**Индексы:**
- `UNIQUE (name)`

**Внешние ключи:**
- `FK (parent_id) → regions.id ON DELETE CASCADE`

### Таблица: `region_audit_logs`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | BIGINT | Первичный ключ, autoincrement |
| `region_id` | BIGINT | FK на regions.id (ON DELETE CASCADE) |
| `action` | VARCHAR(50) | Действие (create, update, delete) |
| `old_data` | JSON | Старые данные (nullable) |
| `new_data` | JSON | Новые данные (nullable) |
| `user_id` | BIGINT | ID пользователя |
| `fio` | VARCHAR(255) | ФИО пользователя |
| `created_at` | TIMESTAMP WITH TIME ZONE | Дата создания |
| `updated_at` | TIMESTAMP WITH TIME ZONE | Дата обновления |

### Таблица: `products` (изменения)

| Поле | Тип | Описание |
|------|-----|----------|
| `region_id` | BIGINT | FK на regions.id (ON DELETE SET NULL, nullable) |

**Индексы:**
- `IX products_region_id`

## 6. Permissions

| Permission | Описание |
|------------|----------|
| `region:create` | Создание нового региона |
| `region:view` | Просмотр регионов (получение по ID, список, дочерние) |
| `region:update` | Обновление региона |
| `region:delete` | Удаление региона |
| `region:audit` | Просмотр аудит-логов |

---

## 7. Примечания

1. **Уникальность имени:** Имя региона должно быть уникальным в рамках всей таблицы (без учёта иерархии).
2. **Минимальная длина имени:** 2 символа (после trim).
3. **Каскадное удаление:** При удалении родительского региона все дочерние регионы удаляются автоматически.
4. **Связь с товарами:** При удалении региона `region_id` у товаров устанавливается в `NULL` (ON DELETE SET NULL).
5. **Иерархия неограниченной глубины:** Поддерживается произвольный уровень вложенности регионов.
6. **Audit-логирование:** Все операции создания, обновления и удаления записываются в `region_audit_logs`.
